2. 正文内容

2.1 DatabaseConnector.java

这个类是整个系统与数据库通信的”桥梁”，负责建立JDBC连接。

  • 首先导入三个包：Connection负责与数据库对接和后续SQL操作；DriverManager用于加载数据库驱动并通过getConnection()方法创建与数据库的连接；SQLException负责抛出SQL异常。
  • 接着先定义数据库的URL，用户名和密码，用于连接数据库；定义getConnection方法用于获取数据库连接对象。
  • 使用Class.forName显式加载 MySQL 驱动程序；加载驱动后，DriverManager才能识别jdbc:mysql://协议（jdbc协议代表使用MySQL驱动）；最后调用DriverManager的静态方法建立数据库连接（一个Connection对象，用于执行SQL语句），这样就顺利连接到了数据库。
  • 代码中使用了try-with-resources 语法，它可以在代码执行结束后自动关闭数据库连接，避免资源泄漏；而throws SQLException 将异常抛出，交由调用者统一处理。

2.2 Student.java

这个类主要用于封装学生的信息，以及增删查改等具体数据库操作方法。

  • 首先导入三个包：java.sql包下所有类（Statement, PreparedStatement, ResultSet等），用于JDBC操作；导入ArrayList，用于存储学生列表；导入List接口，用于定义返回类型；
  • 定义Student类，用于表示学生实体及其数据库操作；定义学生实体的四个属性；定义有参构造方法，用于创建Student对象实例。
  • 定义Getter方法，这些getter方法会在数据库操作中使用；重写Object类的toString方法，方便调试时打印对象信息。
  • 最后分别定义增删查改方法。
  • 特别注意在“增添学生”这个方法中，定义query字符串只有三个占位符，因为id在建表时设置为了自增主键，我们不需要再去设置id，按顺序设置SQL占位符的值。索引从1开始（不是0）。executeUpdate()用于执行INSERT/UPDATE/DELETE语句，返回受影响的行数。
  • 在“查询学生”这个方法中，首先用List存储查询结果，初始为空，后面的ArrayList代表具体实现；ResultSet为结果集对象，像一个指针，可以逐行遍历查询结果；后面的while语句遍历结果集，将每一行数据转换为Student对象，添加到列表中。

2.3 StudentController.java

这是整个项目最核心的类，作为REST API控制器，连接前端和数据库。

  • 首先导入几个包，
  • 注解@RestController，等于 @Controller + @ResponseBody两个注解组合；@Controller：标记这是一个Spring MVC控制器，会被Spring容器管理，@ResponseBody：表示方法返回值直接作为HTTP响应体（自动转JSON），而不是视图名。
  • 注解@RequestMapping(“/api/students”)，定义这个控制器的基础URL路径，该控制器所有方法的URL都以 /api/students 开头，例如：GET /api/students 获取所有学生。
  • 注解@CrossOrigin(origins = “*”)，解决跨域问题，因为前端服务器和后端服务器可能在不同的端口，浏览器默认会阻止，origins = "*" 表示允许来自任何域的请求访问，可以解决这个问题。
  • 注解@GetMapping，标记这个方法处理 HTTP GET 请求，相当于 @RequestMapping(method = RequestMethod.GET)；
  • 其余的如@PostMapping注解，类似的表示标记这个方法处理 HTTP GET 请求；以添加学生为例，当用户在前端页面点击“添加学生”按钮，前端由JS事件监听器’submit’定位到handleFormSubmit方法，接着验证表单，由addStudent接受StudentData数据，发送POST请求给后端；请求到达服务器后，Spring Boot框架会进行路由匹配：由注解@RestController找到RESTAPI控制器；由注解@RequestMapping让spring知道所有以这个URL路径开头的请求都由这个类处理；因为请求是 POST /api/students，Spring会匹配到@PostMapping，由这个addStudent方法参数验证没问题后，会通过Connection conn = DatabaseConnector.getConnection()获取数据库连接；接着创建Studnet对象，调用Studnet类的静态方法将学生插入；最后调用辅助方法createSuccessResponse()创建响应体，Spring会自动把Map序列化为JSON,最后由前端处理响应，返回给客户端。

3. 总结

我学到了什么？

关于JDBC

  • 详细总结Connection类在JDBC中扮演什么角色？
    Connection是JDBC中最核心的接口之一，它代表Java程序与数据库之间的一条通信通道。(StudentController类 第31行，conn就是“通道”，通过conn对象，Java就可以向数据库发送SQL并获取结果)

Connection的主要职责：

  1. 建立连接 -> 与数据库服务器建立TCP连接 -> DriverManager.getConnection()
  2. 创建语句对象 -> 生成执行SQL的Statement或PreparedStatement -> conn.prepareStatement(sql)
  3. 事务管理 -> 控制事务的提交和回滚 -> setAutoCommit(), commit(), rollback()
  4. 资源释放 -> 关闭连接，释放数据库资源 -> close()
  • getConnection()方法在JDBC中扮演什么角色？
    getConnection()是获取数据库连接的入口方法，它负责根据提供的信息建立与数据库的连接。

  • PreparedStatement和Statement的区别？
    Statement：SQL编写用拼接字符串；SQL注入有风险；每次都要重新编译；可读性差；
    PreparedStatement：SQL编写使用?占位符；SQL注入安全；预编译，可复用；可读性好；

  • 详细总结ResultSet是怎么遍历和存储结果的？
    ResultSet是查询结果的容器，它像一个”游标”指向结果集。

  1. rs.next() - 移动游标到下一行，返回是否有数据
  2. rs.getXxx(“列名”) - 获取当前行指定列的值
  3. ResultSet不是把所有数据一次性加载，而是按需从数据库获取

REST是一种设计风格：
HTTP方法 -> 对应操作 -> 我项目中的URL -> 对应方法
GET -> 查询 -> api/students -> getAllStudents()
POST -> 新增 -> /api/students -> addStudent()
PUT -> 修改 -> /api/students/{id} -> updateStudent()
DELETE -> 删除 -> /api/students/{id} -> deleteStudent()

各注解的自动处理：
注解 -> Spring Boot自动做的事
@RestController -> 注册为控制器Bean，返回值自动转JSON
@RequestMapping -> 建立URL到控制器的映射
@GetMapping/@PostMapping -> 建立HTTP方法到具体方法的映射
@RequestBody -> 读取请求体，JSON→Java对象
@PathVariable -> 从URL路径提取参数
@CrossOrigin -> 添加CORS响应头
