《苍穹外卖》解决微信登录模块

发表于2026-01-25|更新于2026-02-10|项目记录
|浏览量:

1. 背景

微信小程序点击登录后，login失败，无法正常登录，无法加载菜品页面。

2. 正文内容

问题的根源是：@AutoFill 注解是为管理端的实体类设计的（如 Employee、Category 等），这些实体类有 createUser 和 updateUser 字段。但 User 是 C端用户实体，它没有这两个字段。

问题根源详解
┌─────────────────────────────────────────────────────────────────────────┐
│ User 实体类只有： │
│ - id, openid, name, phone, sex, idNumber, avatar, createTime │
│ │
│ AutoFillAspect 尝试调用： │
│ - setCreateUser() ← ❌ User 没有这个方法！ │
│ - setUpdateUser() ← ❌ User 没有这个方法！ │
│ - setCreateTime() ← ✅ 有 │
│ - setUpdateTime() ← ❌ User 没有这个方法！ │
└─────────────────────────────────────────────────────────────────────────┘

2.1 核心知识

  • @AutoFill 是之前写的公共字段自动填充功能，它会尝试调用实体类的 setCreateUser()和setUpdateUser()方法。

  • @AutoFill 注解是为管理端实体设计的（如 Employee、Dish、Category），它们有 createUser、updateUser、createTime、updateTime 四个公共字段。但 User 是 C端用户，微信用户自己注册的，不需要记录”谁创建了这个用户”。

  • ✅ 解决方案：移除 @AutoFill 注解
    从 UserMapper.insert() 方法上移除 @AutoFill 注解即可

2.3 主要过程

  • 问题分析与排查思路：

    • 请求已经到达了 UserController.login() 方法（断点能触发），但没有得到 response。这说明问题很可能出在以下几个环节：
    1. 微信 API 调用失败，openid 返回 null；即 getOpenid() 方法调用微信接口可能失败，但代码没有打印返回结果。
    2. 抛出异常但被吞掉；LoginFailedException 被抛出后，全局异常处理器返回了错误，但小程序没正确处理。
    3. code 过期或重复使用，微信的 js_code 是一次性的，5分钟内有效，重复使用会失败。

3. 总结

我学到了什么？

IDEA中debug常用快捷键：

  • F8 Step Over:跳过，一行一行的向下走，如果当前行有方法不进入方法。
  • F7 Step Into：跳入，如果当前有方法，进入方法内部，一般用于进入自定义的方法，不会jdk类库的方法。
  • Shift+ F8 Step Out:跳出，从跳入的方法内退出到方法调用处，这个时候方法已经执行完毕，只是还没有完成赋值
  • Alt+Shift+F7 Force Step Into :强制跳入，可以进入任何方法，想查看底层源代码可以用这个进入jdk的方法。
  • F9 Resume Program:恢复程序，比如，你在第3行和5行各自都打了断点，当前运行到第3行，按F9,则运行到下一个断点即第5行，再按F9，则运行完整个流程，因为后面已经没有断点了
  • Ctrl+First+ F8 View Breakpoints:查看你设置的所有断点。
