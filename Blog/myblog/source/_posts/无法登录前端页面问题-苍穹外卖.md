## 2. 正文内容
    
   #### 关于nginx     
   - nginx是一个高性能的HTTP和反向代理服务器，也是一个IMAP/POP3/SMTP代理服务器，常用于静态内容服务，负载均衡，反向代理等。
   - 1. **前端发送的请求如何传送到后端？——nginx反向代理**
    前端请求地址：http://localhost/api/employee/login   后端接口地址：http://localhost:8080/admin/employee/login
    可见前后端的请求地址并不匹配，但它们的确请求成功了。**nginx反向代理，将前端的动态请求由nginx转发给后端的服务器。**
    （前端首先请求nginx服务器，nginx再转发给后端服务器）  可为什么前端不直接请求后端呢？ => 
    nginx的作用：
    - 提高访问速度
    - 进行负载均衡：由于前端只可以固定的访问某一台后端服务器，当访问请求较大时，负载均衡就是让nginx可以均匀的把请求分配给每一个后端服务器。
    - 保证后端服务安全：后端的端口一般不会直接暴露，它存在于公司局域网内，前端要发送请求只可以通过nginx走内网发送。

### 2.2 nginx反向代理配置
```properties
    server {
        listen       80;
        server_name  localhost;

        location /api/ {
            proxy_pass: http://localhost:8080/admin/;
        }
    }
```

### 2.3 nginx负载均衡配置
```properties
    upstream webservers{
	  server 127.0.0.1:8080 weight=90 ;  //声明两个服务器，weight是权重，越高所分配的客户端请求越多。
	  #server 127.0.0.1:8088 weight=10 ;
	}

    server {
        listen       80;
        server_name  localhost;

        location /api/ {
            proxy_pass: http://webservers/admin/; //webservers是随机的名称，确保和上面的upstream保持一致既可。
        }
    }
```
